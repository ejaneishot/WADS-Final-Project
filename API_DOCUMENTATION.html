Authentication

All protected routes require a valid JWT stored in an HTTP-only cookie named smartcareer_token. The cookie is set automatically by the login and register endpoints — browsers handle it transparently.

For API testing tools (Postman, curl), copy the cookie after logging in and pass it manually:

Cookie: smartcareer_token=<your-jwt-token>
🔓 Public No authentication required    🔒 Auth Valid JWT cookie (student or admin)    🛡️ Admin Admin role required    ✨ AI Calls Google Gemini

1. Authentication Endpoints

POST /api/auth/register 🔓 Public

Register a new account. All accounts are created with the student role — admin accounts cannot be self-registered via this endpoint.

Request body

{ "email": "student@example.com", "password": "SecurePass123" }
FieldTypeRules
emailstringValid email format
passwordstring8–72 characters

Response 201 Created

{ "ok": true, "user": { "id": "clx1abc123", "email": "student@example.com", "role": "student" } }

Error responses

StatusBodyReason
400{ "message": "Invalid input", "issues": [...] }Zod validation failed
409{ "message": "Email already registered" }Duplicate email
POST /api/auth/login 🔓 Public

Login with email and password. Sets the smartcareer_token cookie on success.

Request body

{ "email": "student@example.com", "password": "SecurePass123" }

Response 200 OK

{ "ok": true, "user": { "id": "clx1abc123", "email": "student@example.com", "role": "student" } }

Error responses

StatusBodyReason
400{ "message": "Invalid input" }Validation failed
401{ "message": "Invalid credentials" }Wrong email or password
POST /api/auth/logout 🔓 Public

Clears the smartcareer_token cookie (sets maxAge: 0).

Response 200 OK

{ "ok": true }
POST /api/auth/login-google 🔓 Public

Exchange a Firebase Google ID token for a local JWT session cookie. The server verifies the Firebase token using Firebase Admin SDK.

Request body

{ "idToken": "<firebase-google-id-token>" }

Response 200 OK

{ "ok": true, "user": { "id": "clx1abc123", "email": "student@gmail.com", "role": "student" } }

Error responses

StatusBodyReason
400{ "message": "Missing idToken" }No token in body
401{ "message": "Invalid Google token" }Firebase verification failed

Returns all quiz sections with their questions and answer options. Use this to render the assessment UI.

Response 200 OK

[{
  "id": "sec_1", "title": "Work Style", "description": "How do you prefer to work?", "order": 1,
  "questions": [{
    "id": "q_1", "prompt": "Which environment excites you most?",
    "helperText": "Choose what feels most natural.", "type": "SINGLE_CHOICE",
    "isRequired": true, "order": 1,
    "options": [{ "id": "opt_1", "label": "Building user interfaces", "value": "fe", "order": 1 }]
  }]
}]
POST /api/assessment/submit 🔒 Auth

Submit quiz answers. The server cross-validates every optionId against its questionId, computes weighted career-tag scores, and persists the top-two result.

Request body

{
  "answers": [
    { "questionId": "q_1", "optionId": "opt_1" },
    { "questionId": "q_2", "optionId": "opt_5" }
  ]
}
FieldRules
answersNon-empty array; no duplicate questionId values
optionIdMust exist in DB and belong to its questionId

Response 200 OK

{
  "ok": true, "attemptId": "att_xyz",
  "createdAt": "2026-06-01T10:00:00.000Z",
  "result": { "primary": "SWE", "secondary": "AI" }
}

Error responses

StatusBodyReason
400{ "message": "Invalid input" }Zod validation failed
400{ "message": "Duplicate questionId in answers payload" }Same question answered twice
400{ "message": "One or more options not found" }Invalid optionId
400{ "message": "optionId does not belong to questionId" }Cross-question mismatch
500{ "message": "Failed to compute result" }All options had zero weight
GET /api/assessment/result 🔒 Auth

Returns the most recent assessment attempt for the authenticated user.

Response 200 OK (has result)

{ "attemptId": "att_xyz", "primary": "SWE", "secondary": "AI", "scores": { "SWE": 12, "AI": 8, "FE": 3 }, "createdAt": "2026-06-01T10:00:00.000Z" }

Response 200 OK (no attempt yet)

{ "attempt": null }

POST /api/roadmaps/[id]/generate 🔒 Auth ✨ AI

Generate 6–8 learning nodes for a roadmap using Google Gemini. A two-stage guard runs first: (1) topic classifier rejects non-tech topics and prompt injections, (2) ownership check confirms the roadmap belongs to the caller. Generation can optionally be personalized from the user's profile, skills, and latest assessment result.

Request body

{ "topic": "Master Kubernetes", "includeProfile": true }
FieldTypeRules
topicstring3–800 characters; must be a tech career learning goal
includeProfilebooleanOptional. Personalizes output from user profile + assessment

Response 200 OK

{ "success": true }
Nodes are persisted directly to the database. Re-fetch GET /api/roadmaps/[id] to retrieve the generated nodes and edges.

Error responses

StatusBodyReason

6. Resume Optimizer Endpoints

POST /api/upload 🔓 Public

Upload a PDF resume file and extract its plain text content. Pass the returned text field to POST /api/analyze.

Request: multipart/form-data

FieldType
filePDF file (binary)

Response 200 OK

{ "text": "Anastasia Larasati\nSoftware Engineer\nBINUS University..." }

Error responses

StatusBodyReason
400{ "message": "No file uploaded" }Missing file field
400{ "message": "Only PDF files are supported" }Wrong file type
POST /api/analyze 🔓 Public ✨ AI

Analyze resume text with Google Gemini. Runs a classifier first to confirm the input is a resume or resume draft, then performs a full expert recruiter analysis. Prompt injection via the input field is blocked by the classifier guard.

Request body

{ "text": "Anastasia Larasati\nSoftware Engineer\nBuilt REST APIs using Node.js..." }
FieldRules
textNon-empty string; truncated silently at 48,000 characters

Response 200 OK

{
  "good": [{ "phrase": "Built REST APIs using Node.js", "reason": "Specific technology + action verb" }],
  "bad": [{ "phrase": "Responsible for", "reason": "Passive language — replace with an action verb" }],
  "suggestedSkills": ["TypeScript", "Docker", "CI/CD"],
  "criticalImprovements": "Quantify your achievements with metrics. Add a professional summary."
}

Error responses

StatusBodyReason
400{ "message": "Missing or empty \"text\" field" }No text provided
400{ "message": "Your input isn't a resume..." }Classifier rejected input
500{ "message": "Could not validate your resume text" }Classifier returned null
502{ "message": "Empty model response" }Gemini returned empty body
502{ "message": "Model returned invalid JSON" }Gemini returned malformed JSON
503{ "message": "Server is not configured..." }GEMINI_API_KEY not set

7. Code Execution Endpoint

POST /api/code/execute 🔒 Auth

Run learner code in a sandboxed environment. The server proxies the request to the free Wandbox public compile API — no additional API key is required. Used by the interactive lesson pages under /careers/[slug]/learn.

Request body

{ "language": "python", "code": "print('Hello, World!')" }
FieldRules
languageOne of: python, javascript, java, csharp, bash
codeNon-empty string; max 10,000 characters
Language valueCompiler / Runtime
pythonCPython 3.11.10
javascriptNode.js 20.17.0
javaOpenJDK 21
csharpMono 6.12
bashbash

Response 200 OK

{ "stdout": "Hello, World!\n", "stderr": "", "compileError": "", "status": "0" }

Error responses

StatusBodyReason
400{ "message": "Unsupported language" }Language not in allowed list
400{ "message": "Code is required" }Empty code field
400{ "message": "Code is too long" }Exceeds 10,000 character limit
502{ "message": "Code execution service unavailable" }Wandbox returned error
504{ "message": "Code execution timed out" }Wandbox did not respond within 20 s

11. Error Reference

Standard error body

All error responses use this shape:

{ "message": "Human-readable description of the error" }

Validation errors also include an issues array from Zod:

{ "message": "Invalid input", "issues": [{ "path": ["email"], "message": "Invalid email" }] }

HTTP status codes

StatusMeaningCommon cause
400Bad RequestValidation failed, missing fields, business rule violation
401UnauthorizedNo JWT cookie, or invalid / expired token
403ForbiddenValid JWT but insufficient role; or CSRF origin mismatch
404Not FoundResource does not exist or does not belong to the user
409ConflictDuplicate resource (e.g. email already registered)
500Internal Server ErrorUnexpected server or database error
502Bad GatewayGemini or Remotive returned an error or invalid response
503Service UnavailableRequired server config missing (e.g. GEMINI_API_KEY)
504Gateway TimeoutExternal service (Wandbox, Remotive) did not respond in time
